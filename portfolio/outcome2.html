Network used for the internet

Installing DHCP server on Ubuntu Server 16.04

This was in open source class, with Curtis Gabrielson as the instructor. I used VM (Virtual Machine) to install the DHCP. Examples of the steps that were included; I installed isc-dhcp-server package using the apt commands, and then I enabled the firewall, modifying the interfaces to specify which Ethernet interface to use. Lastly, I edited the dhcp/dhcp.d configuration files, adding the subnets, netmasks and IP ranges. What I learned from this lab is that an administrator can change the IP addresses of a larger number of system, they can also configure just one DHCP file on a server for new set of IP addresses instead of configuring the entire system. Being observant and thorough will help one avoid some of the minor and petty mistakes in the configuration files that can save time on that part. The problem that I ran into is leaving my laptop Wi-Fi on which interfered with the bridged network and made it so I could not use putty to shh into my server. I may be a web developer but it is very important for me to know and understand how the network system works and how to troubleshoot these simple yet frustrating problems.





Installation, configuring and testing of Bind9 DNS server

This also happened in Open source class with Curt as the instructor. The steps that I took included the installation of BIND9, as well as the configuration of the DNS forwarders, and then I configured the zone declarations, which specified the location of my db.example.com file and contains its zone information. The last step, which I found to be interesting, was the reverse lookup zone, the reason I found this interesting was that it holds the reverse IP and can search for them based off it. What I learned from this lab how networks allow for individual replacement of IP addresses of individual machines by a name, and how to associate multiple names to the same machine to update the different available services. The problem I faced during this lab was my isc-dhcp-server was not active at a point where it should have been and because of this, I could not resolve the hostname of my servers. Therefore, there was errors in the configuration file that I had to add the subnets and IP ranges. All I needed in this lab is to stay focused and motivated to finish the lab. Once again, it is important to know how to implement a network properly and that it does not matter if you are a developer you should still know this.
